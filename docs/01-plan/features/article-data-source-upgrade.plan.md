## 1. 문제 진단: 현재 PDF 기반 법조항 데이터의 품질 이슈

### 1.1 이전 PDCA 아카이브 분석 결과

5개 완료된 PDCA 사이클에서 반복적으로 발견된 문제:

| PDCA Cycle | Match Rate | 발견된 데이터 관련 이슈 |
|-----------|-----------|----------------------|
| ohs-article-matching (#1) | 92% | physical 카테고리 범위 너무 넓음 (32~166), 편/장/절 메타데이터 없음 |
| kosha-guide-mapping (#2) | 87% | 법조항 47% 미매핑, 가이드 33% 미매핑 |
| kosha-guide-matching-v2 (#3) | 93% | GPT 키워드 비결정론성, explicit mapping 정확도 한계 |
| analysis-ontology-integration (#4) | 90% | physical 하위분류 필요, 시맨틱 정확도 부족 |
| matching-quality-v2 (진행중) | - | 임베딩 한계, 쿼리 품질, 법률용어 갭 |

**공통 근본 원인**: 법조항 원본 데이터 품질 자체가 낮음

### 1.2 현재 데이터 품질 정량 분석

```
법조항 데이터 현황 (articles_cache.json + ChromaDB)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
총 조문 수:         442건 (제1조 ~ 제673조)
고유 조문 번호:      435건 (7건 중복/변형)
누락 조문 범위:      174개 gap range
평균 내용 길이:      439자 (원문 대비 매우 짧음)
PDF 아티팩트 포함:   254건 (57.5%) ← 심각
내용 잘림(truncated): 전체적으로 만연
50자 미만 조문:      4건 (사실상 빈 데이터)
```

### 1.3 구체적 품질 문제

#### A. PDF 아티팩트 오염 (57.5%)
```
# 제2조 content 끝부분 예시
"...25년-1총칙-10차 복사본.indd   10\n2025. 3. 4.   오후 5:21"
```
→ InDesign 파일명, 타임스탬프가 법조항 텍스트에 포함
→ 벡터 임베딩 시 노이즈로 작용하여 검색 정확도 저하

#### B. 내용 잘림 (Truncation)
```
# 제42조 (추락의 방지) - 원문은 4개 항인데 content가 중간에 끊김
"...필요한 조치를 해야 한"  ← 문장 중간에서 절단
# 제301조 (충전부 방호) - 핵심 조건절 누락
"...접근"  ← 문장 중간에서 절단
```
→ PDF 페이지 경계에서 텍스트 추출이 끊긴 결과
→ 핵심 법적 요건이 누락되어 벡터검색 및 키워드 매칭 정확도 저하

#### C. 메타데이터 부재
```
현재: { article_number, title, content, source_file }
없는 것: 편(Part), 장(Chapter), 절(Section), 조의2 계층,
         시행일, 개정이력, 항(paragraph) 단위 분리
```
→ 카테고리 필터링 시 편/장/절 기반 범위지정 불가
→ 항(①②③) 단위 정밀 매칭 불가

#### D. 누락 조문 (174개 gap range)
- 삭제 조문이 아닌 실제 존재 조문이 빠져있을 가능성
- PDF 파일 자체가 없거나, 파싱 실패한 것으로 추정

## 2. 해결 방안 비교

### Option A: law.go.kr Open API 활용 (권장)

**접근 방식**: 국가법령정보센터 Open API로 조문 단위 구조화된 데이터 수집

```
API Endpoint: http://www.law.go.kr/DRF/lawService.do
Parameters:
  OC=<등록ID>       # IP/도메인 등록 필요 (02-2109-6446)
  target=lawjosub   # 조항호목 조회
  type=XML|JSON     # 응답 형식
  ID=007363         # 산업안전보건기준에 관한 규칙
  JO=000300         # 조문번호 6자리 (제3조 = 000300)
```

**응답 데이터 구조 (XML)**:
```xml
<법령>
  <조문>
    <조문번호>제3조</조문번호>
    <조문제목>전도의 방지</조문제목>
    <조문내용>사업주는 근로자가 작업장에서 넘어지거나...</조문내용>
    <항>
      <항번호>①</항번호>
      <항내용>사업주는 근로자가...</항내용>
      <호>
        <호번호>1.</호번호>
        <호내용>...</호내용>
      </호>
    </항>
  </조문>
</법령>
```

| 장점 | 단점 |
|------|------|
| 조문/항/호/목 계층 구조 완벽 보존 | API 키(IP 등록) 필요 |
| PDF 아티팩트 0% (공식 텍스트) | 전화 등록 절차 필요 (02-2109-6446) |
| 내용 잘림 없음 (전문 제공) | API 가용성 의존 |
| 최신 개정본 자동 반영 | 초기 전량 수집 시 트래픽 부담 가능 |
| 편/장/절 메타데이터 제공 | |
| 시행일/개정이력 포함 | |

### Option B: law.go.kr 웹 스크래핑 (Playwright)

**접근 방식**: 웹브라우저 자동화로 법조항 텍스트 직접 수집

```
URL 패턴: https://www.law.go.kr/법령/산업안전보건기준에관한규칙/제{N}조
→ Playwright로 렌더링 후 조문 텍스트 추출
```

| 장점 | 단점 |
|------|------|
| API 키 불필요 | JavaScript 렌더링 필요 (Playwright) |
| 바로 시작 가능 | 스크래핑 차단 위험 (robots.txt, rate limit) |
| 편/장/절 구조 추출 가능 | 유지보수 부담 (HTML 구조 변경 시 깨짐) |
| | 법적 이슈 가능 (이용약관) |

### Option C: 기존 PDF 데이터 정제 (최소한의 수정)

**접근 방식**: 현재 데이터의 아티팩트 제거 + 잘린 내용 보완

```python
# 아티팩트 제거 regex
cleaned = re.sub(r'\d{2}년-.*?\.indd.*$', '', content, flags=re.MULTILINE)
cleaned = re.sub(r'\d{4}\.\s*\d{1,2}\.\s*\d{1,2}\..*?(오전|오후).*$', '', cleaned, flags=re.MULTILINE)
```

| 장점 | 단점 |
|------|------|
| 가장 빠르게 적용 가능 | 잘린 내용 복구 불가 |
| 기존 파이프라인 수정 최소 | 누락 조문 보완 불가 |
| | 항/호/목 구조화 불가 |
| | 근본 해결이 아님 |

### Option D: 하이브리드 (C 즉시 적용 → A 순차 진행)

**접근 방식**: 즉시 PDF 정제(Option C) 적용 후, API 등록 완료되면 전면 교체(Option A)

## 3. 권장안: Option D (하이브리드)

### Phase 1: PDF 데이터 즉시 정제 (1시간)

**목표**: 57.5% PDF 아티팩트 제거로 당장의 검색 품질 개선

1. `articles_cache.json` 정제 스크립트 작성
   - InDesign 아티팩트 제거 (`.indd`, 타임스탬프)
   - 불필요한 공백/개행 정리
2. ChromaDB 재인덱싱 (정제된 데이터로)
3. 매칭 테스트 재실행

### Phase 2: law.go.kr API 등록 및 전면 교체

**목표**: 완전한 구조화 데이터로 근본 전환

- 전화: 02-2109-6446
- 서버 IP 등록 요청
- 예상 소요: 1~3 영업일

```python
        # rate limiting 적용 (0.5초 간격)
```

#### Step 2-3: 데이터 스키마 확장
```python
# 현재 스키마
{"article_number": "제3조", "title": "전도의 방지", "content": "..."}

# 확장 스키마
{
    "article_number": "제3조",
    "title": "전도의 방지",
    "part": "제2편 안전기준",           # 편
    "chapter": "제1장 총칙",            # 장
    "section": null,                    # 절
    "full_content": "...(전문)...",     # 잘림 없는 전문
    "paragraphs": [                     # 항 단위 분리
        {"number": "①", "content": "...", "items": [...]},
        {"number": "②", "content": "...", "items": [...]}
    ],
    "enforcement_date": "2024-01-01",   # 시행일
    "revision_info": "...",             # 개정 정보
    "is_deleted": false                 # 삭제 조문 여부
}
```

#### Step 2-4: ChromaDB 재인덱싱 전략
```
현재: 1 article = 1 embedding (442건)
개선: 1 paragraph = 1 embedding (예상 ~1,500건)
     + 1 article summary = 1 embedding (전체 요약)
```
- **항(paragraph) 단위 임베딩**으로 검색 정밀도 대폭 향상
- 항 단위 → 조문 → 편/장/절 역추적으로 계층적 매칭 가능

#### Step 2-5: KOSHA 가이드 매핑 개선
```
현재: 법조항 제목 + 짧은 content ↔ KOSHA GUIDE 시맨틱 매핑
개선: 법조항 전문(항 단위) ↔ KOSHA GUIDE 섹션 단위 정밀 매핑
```
- 전문 데이터로 매핑 커버리지 47% → 70%+ 기대
- 항 단위 매핑으로 더 구체적인 법적 근거 제시 가능

## 8. 결론

**PDF 분할 방식의 근본 한계**: 현재 442건의 법조항 데이터는 PDF에서 추출한 것으로, 57.5%에 아티팩트가 포함되고 내용이 잘리는 등 **데이터 자체의 품질이 매칭 정확도의 병목**입니다.

**law.go.kr API 전환 권장**: 국가법령정보센터 Open API를 통해 조문/항/호/목 단위의 완전한 구조화 데이터를 수집하면:
- 아티팩트 0%, 잘림 0%
- 항 단위 정밀 매칭 가능
- 편/장/절 메타데이터로 카테고리 필터링 정확도 향상
- KOSHA 가이드 매핑 커버리지 확대

**즉시 적용**: API 등록 대기 중에도 Phase 1(PDF 정제)으로 당장의 품질 개선 가능
