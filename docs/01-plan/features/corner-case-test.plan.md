# Plan: 코너케이스 대량 테스트 (OpenAI API 미사용)

## 배경

현재 매칭 성능:
- **법령 조문**: 81.6% (40/49) - GPT-4.1 생성 시나리오
- **KOSHA GUIDE**: 34% 정확, 64% 분류 - GPT-4.1 생성 시나리오

문제점:
1. 기존 테스트 데이터가 OpenAI API(GPT-4.1)로 생성되어 비용 발생
2. GPT 생성 시나리오가 너무 일반적 → 코너케이스 부족
3. 실패 패턴 분석 없이 무작위 샘플링

## 목표

- OpenAI API 없이 Claude가 직접 테스트 데이터 생성
- 코너케이스 중심 100개 시나리오 (법령 50 + KOSHA 50)
- 현재 취약 영역 집중 공략

## 코너케이스 유형 설계

### A. 법령 조문 코너케이스 (50개)

| 카테고리 | 개수 | 설명 |
|----------|------|------|
| 1. 기존 실패 조문 재도전 | 8 | 제24조, 제34조, 제49조, 제63조, 제73조, 제79조, 제611조, 제660조 |
| 2. 복합 위험 시나리오 | 8 | 추락+감전, 화재+질식, 끼임+화상 등 2개 이상 위험 겹치는 상황 |
| 3. 근접 조문 구별 | 8 | 제42조 vs 제43조 vs 제44조, 제301조 vs 제304조 vs 제319조 등 |
| 4. 비전형 작업환경 | 6 | 농업, 선박, 광산, 연구실, 병원, 사무실 |
| 5. 관리/행정 조항 | 6 | 통지의무, 교육, 점검, 기록유지 등 |
| 6. 설비 규격 조항 | 6 | 특정 설비(보일러, 압력용기, 덕트, 사다리) |
| 7. 드문 위험 유형 | 4 | 방사선, 석면, 벌목, 잠수 |
| 8. 모호한 표현 시나리오 | 4 | 법적 용어 없이 일상 언어로 묘사 |

### B. KOSHA GUIDE 코너케이스 (50개)

| 카테고리 | 개수 | 설명 |
|----------|------|------|
| 1. 기존 실패 가이드 재도전 | 10 | E분류(0%), G분류(33%) 중심 |
| 2. 화학물질명 직접 언급 | 8 | A분류 정확 매칭 강화 |
| 3. 기계/설비 구체적 명시 | 8 | B, M분류 정확 매칭 |
| 4. 공정 안전 시나리오 | 6 | C, P분류 |
| 5. 보건관리/건강장해 | 6 | H분류 (현재 29% 정확) |
| 6. 비주류 분류 | 6 | F, K, O, W, X 분류 |
| 7. 복합 분류 시나리오 | 6 | 여러 분류가 겹칠 수 있는 상황 |

## 구현 계획

### Step 1: 테스트 데이터 생성 (Claude 직접 작성)
- `data/corner_test_articles_50.json` - 법령 조문 코너케이스 50개
- `data/corner_test_kosha_50.json` - KOSHA GUIDE 코너케이스 50개
- JSON 포맷은 기존 `bulk_test_50.json`, `kosha_test_50.json`과 동일

### Step 2: 테스트 실행
- 기존 `run_bulk_test.py`, `run_kosha_test.py` 재활용
- 테스트 데이터 경로만 파라미터로 변경

### Step 3: 결과 분석
- 코너케이스별 매칭률 집계
- 실패 패턴 분류
- 개선 방향 도출
