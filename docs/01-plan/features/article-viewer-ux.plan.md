# Plan: article-viewer-ux

> 관련 법조항 보기 UI/UX 개선 — 끊어진 원문 링크 대체 + 사용자 경험 향상

## 1. 현재 상태 (AS-IS)

### 1.1 문제

```
사용자 플로우:
  결과 페이지 → "안전지침 & 법조항" 탭 → KOSHA 가이드 펼침 → "관련 법조항" 목록
    → "원문 →" 클릭 → 💥 404 에러 (링크 끊김)
```

**원인**: `source_file`이 `"law.go.kr (lsiSeq=280187)"`인데, 프론트엔드가 `/articles-pdf/{source_file}` URL로 PDF 서빙을 시도. 백엔드에 해당 엔드포인트 없음.

### 1.2 현재 UI 구조

```
RelatedGuides.tsx
├── KOSHA 가이드 카드 (접기/펼기)
│   ├── 가이드 코드 + 분류 배지 + 법조항 N건
│   ├── [펼침] 관련 섹션 (section_title + excerpt)
│   └── [펼침] 관련 법조항 목록
│       ├── 제42조 (추락의 방지) | 원문 → ← 💥 깨짐
│       └── 제63조 (비계 작업)   | 원문 → ← 💥 깨짐
```

### 1.3 사용할 수 있는 데이터

| 필드 | 현재 전달량 | 백엔드 보유량 | 비고 |
|------|:-----------:|:------------:|------|
| `article_number` | 전체 | - | 제42조, 제63조 등 |
| `title` | 전체 | - | 추락의 방지 등 |
| `content` | 300자 | 최대 3000자 | `analysis_service.py:449`에서 `[:300]` |
| `source_file` | 전체 | - | `"law.go.kr (lsiSeq=280187)"` (무의미) |
| `chapter` | 미전달 | ChromaDB metadata | 장 정보 (편/장 구조) |
| `hazard_major` | 미전달 | taxonomy.py | 위험 대분류 |
| `hazard_codes` | 미전달 | taxonomy.py | 세부 위험 코드 |

## 2. UI/UX 개선안 비교

### 안 A: law.go.kr 원문 링크 (최소 변경)

```

```

- "원문 →" 링크를 law.go.kr 조문 페이지로 교체
- URL: `https://www.law.go.kr/LSW/lsInfoP.do?lsiSeq=280187#제42조`
- **장점**: 프론트 1줄 수정, 공식 원문, 즉시 적용 가능

## 3. 추천: 안 B + 안 D 조합 (하이브리드)

### 핵심 아이디어

```
기본 상태: 안 B (인라인 미리보기)
  → 법조항 클릭 시 content 첫 2줄 미리보기 + "전문 보기" 버튼

전문 보기: 안 D (모달)
  → "전문 보기" 클릭 시 모달로 전체 내용 + 메타데이터
```

### 구현 범위

**프론트엔드 (3개 파일)**

| 파일 | 변경 |
|------|------|
| `RelatedGuides.tsx` | "원문 →" 제거, 인라인 미리보기 + 모달 트리거 |
| `ArticleModal.tsx` | **신규** — 조문 전문 모달 컴포넌트 |
| `types/analysis.ts` | `GuideArticleRef`에 `chapter`, `hazard_major` 추가 |

**백엔드 (2개 파일)**

| 파일 | 변경 |
|------|------|
| `analysis_service.py` | `content[:300]` → `content` 전문 전달, `chapter`/`hazard_major` 추가 |
| *(선택)* 별도 API | `GET /api/v1/articles/{article_number}` 조문 상세 엔드포인트 |

### 단계별 실행

| 단계 | 내용 | 효과 |
|------|------|------|
| **Step 1** (즉시) | 깨진 "원문 →" → law.go.kr 링크로 교체 | 🔴 깨진 링크 해결 |
| **Step 2** | content 전문 전달 + 인라인 미리보기 | 기본 정보 빠르게 확인 |

## 4. 상세 스펙

### 4.1 law.go.kr 원문 URL 생성 규칙

```
기본 URL: https://www.law.go.kr/LSW/lsInfoP.do?lsiSeq=280187&ancYnChk=0
조문 앵커: #0000 (법률 전체 뷰, 페이지 내에서 Ctrl+F 필요)
```

> law.go.kr은 개별 조문 딥링크를 공식 지원하지 않음.
> 따라서 외부 링크는 보조 수단이고, 인앱 열람이 주요 경로가 되어야 함.

### 4.2 ArticleModal 컴포넌트 스펙

```typescript
interface ArticleModalProps {
  article: {
    article_number: string;  // "제42조"
    title: string;           // "추락의 방지"
    content: string;         // 전문 (최대 3000자)
    chapter?: string;        // "제6장 추락·붕괴 위험 방지"
    part?: string;           // "제2편 안전기준"
    hazard_major?: string;   // "physical"
  };
  isOpen: boolean;
  onClose: () => void;
}
```

**모달 레이아웃:**
1. 헤더: 조문번호 + 제목 + 닫기 버튼
2. 메타데이터 배지: 편/장 위치 + 위험 대분류
3. 본문: 항별 파싱하여 가독성 높게 표시 (①, ②, ③ 구분)
4. 하단: law.go.kr 외부 링크 (보조)

### 4.3 인라인 미리보기 스펙

```
현재:  제42조 (추락의 방지) | 원문 →

개선:  제42조 (추락의 방지)                    [전문 보기]
       사업주는 근로자가 추락하거나 넘어질 위험이 있는 장소에...
```

- content 첫 80자 미리보기 (한 줄)
- 클릭 시 모달 열림
