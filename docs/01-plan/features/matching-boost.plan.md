# 코너케이스 매칭률 개선 (matching-boost)

> PDCA Phase: Plan → Do
> Created: 2026-02-27

## 목표
코너케이스 테스트 결과 발견된 취약점을 4가지 개선으로 해결

### 현재 성능 (코너케이스 기준)
- 법령 조문: 76.0% (38/50) — 설비규격 33%, 관리조항 50%
- KOSHA 정확: 53.1% (26/49) — G 33%, W 0%, H정확 25%
- KOSHA 분류: 75.5% (37/49)

## 개선 항목 (구현 순서)

### 1. 키워드→조문/가이드 매핑 테이블
- `data/keyword_mappings.json` 생성
- 실패 조문/가이드에 대한 직접 키워드 매핑
- `analysis_service.py`에서 키워드 매칭 후 후보에 주입

### 2. GPT 힌트 Few-shot 실패사례 보강
- `analysis_prompts.py`의 핵심 조문 매칭 가이드 테이블에 실패 사례 추가
- 설비규격(제120조, 제174조, 제222조, 제261조) 등 누락 조문 보강

### 3. BM25 하이브리드 검색
- `rank-bm25` 라이브러리 추가
- `article_service.py`, `guide_service.py`에 BM25 인덱스 구축
- 벡터 점수 50% + BM25 점수 50% 하이브리드 스코어링

### 4. KOSHA 분류 사전 라우팅
- 시나리오 키워드로 분류 예측 → 해당 분류 우선 검색
- `guide_service.py`에 분류 키워드 사전 추가

## 성공 기준
- 법령 조문: 76% → 85%+
- KOSHA 정확: 53% → 65%+
- KOSHA 분류: 75% → 85%+
