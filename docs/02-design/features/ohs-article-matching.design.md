## 3. GPT 프롬프트 개선 설계 (FR-06)

### 3.1 시스템 프롬프트에 조문 목차 추가

GPT가 분석 시 관련 조문번호를 **직접 추천**할 수 있도록, 시스템 프롬프트에 주요 조문 목차를 포함합니다.

```python
ARTICLE_INDEX_PROMPT = """
## 산업안전보건기준에 관한 규칙 주요 조문 목차

[제1편 총칙]
- 제3조 전도의 방지 | 제14조 낙하물에 의한 위험의 방지
- 제42조~제52조 추락·붕괴에 의한 위험 방지
- 제53조~제71조 비계

[제2편 안전기준]
- 제86조~제99조 기계 일반기준 (원동기, 회전축, 방호장치)
- 제100조~제102조 공작기계
- 제103조~제104조 프레스·전단기
- 제105조~제109조 목재가공용 기계 (둥근톱, 띠톱)
- 제131조~제170조 양중기 (크레인, 리프트)
- 제171조~제188조 차량계 하역운반 (지게차)
- 제225조~제300조 폭발·화재 및 위험물
- 제301조~제327조 전기로 인한 위험 방지
- 제328조~제378조 건설작업

[제3편 보건기준]
- 제420조~제450조 관리대상 유해물질
- 제511조~제518조 소음·진동
- 제555조~제572조 온도·습도
- 제617조~제644조 밀폐공간

위험요소를 분석할 때, 위 목차를 참고하여 관련 조문번호를 추천해주세요.

## 4. 하이브리드 매칭 파이프라인 설계 (FR-03, FR-04, FR-05)

### 4.1 파이프라인 흐름

```
┌──────────────────────────────────────────────────────────────────┐
│                    하이브리드 매칭 파이프라인                       │
├──────────────────────────────────────────────────────────────────┤

│  INPUT: hazards[] (GPT 분석 결과)                                │
│                                                                  │

│  │         │   │   조문범위)   │   │   + 쿼리개선) │              │

### 4.2 Stage별 상세 설계

#### Stage 1: GPT 직접 추천
- 기존 GPT 분석 응답에 `related_articles` 필드 추가
- 시스템 프롬프트에 조문 목차 포함
- 가중치: **0.4** (가장 높은 신뢰도)

#### Stage 2: 하드매핑
- `category_code` → `CATEGORY_TO_ARTICLES` 딕셔너리 조회
- primary 범위의 조문을 ChromaDB에서 ID로 직접 조회
- 가중치: **0.35**

#### Stage 3: 벡터 검색 (개선)
- 기존 방식 유지하되, 쿼리를 개선
- 쿼리 = `"{카테고리 keywords} {hazard.description}"`
- 가중치: **0.25**

#### Stage 4: 후보 통합
- 3개 Stage 결과를 통합 (중복 조문번호 제거)
- 가중치 합산으로 초기 점수 계산
- 최대 10개 후보 선정

#### Stage 5: LLM Reranker
- GPT-4.1-mini로 후보 10개를 재평가
- 프롬프트: "다음 위험요소에 대해 가장 관련 있는 법조항 5개를 선택하고 관련성 점수를 매겨주세요"
- 최종 상위 5개 반환

### 4.3 Reranker 프롬프트

```python
RERANKER_PROMPT = """다음 위험요소에 대해, 후보 법조항 중 가장 관련 있는 것을 선택하세요.

## 위험요소
{hazard_summary}

## 후보 법조항
{candidates_json}

각 후보에 대해 관련성 점수(0.0~1.0)를 매기고, 상위 5개만 반환하세요.
관련성 기준: 해당 위험요소를 예방·관리하기 위해 사업주가 준수해야 하는 조문인가?
"""
```
